Fast API 入門筆記 (一)

  • 2021- 1-21 (四)
  • By MingLun Allen Wu

Introduction

FastAPI的特色: + 快速： 最快速的Python Framework + 開發快速

Requirements

  • Python 環境 : Python 3.6 +

    :::bash pip install fastapi # FastAPI pip install uvicorn[standard] # ASGI Server

撰寫第一支簡單的API

在語法結構上，與Flask大同小異，透過 @app 裝飾子來定義 API路徑

from typing import Optional

from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"Hello": "World"}


@app.get("/items/{item_id}")
def read_item(item_id: int, q: Optional[str] = None):
    return {"item_id": item_id, "q": q}

與 Flask 不同的是，如果直接執行此 main.py 不會啟動服務。 需要透過 ASGI Server gvicorn 來啟用：

::: bash
uvicorn main:app --reload
# uvicorn <檔名>:<app名稱>

啟用後在瀏覽器輸入 127.0.0.1:8000/items/8?q=minglunwu，將會得到如下的回傳結果: 

{"item_id":8,"q":"minglunwu"}

與 Flask 相同，當我們鍵入 127.0.0.1:8000/items/8?q=minglunwu， FastAPI會自動尋找符合條件的路徑前綴: /items

不同的是：FastAPI 透過 型態提示 (Typing Hints) 來作為接受參數的依據，在定義參數時，需要連同參數的型態一併定義。 在上圖的 read_item() 中，定義了兩個參數：

  • item_id: int格式
  • q: str格式

如果此時我們嘗試在瀏覽器中輸入 不符合規則的URL: 127.0.0.1:8000/items/error_type?q=minglunwu (將原本的 8 換成字串 error_type)，則我們會看到下列訊息:

{"detail":[{"loc":["path","item_id"],"msg":"value is not a valid integer","type":"type_error.integer"}]}
