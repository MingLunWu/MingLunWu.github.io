Fast API 入門筆記 (一)

  • 2021- 1-21 (四)
  • By MingLun Allen Wu

Introduction

FastAPI的特色:

  • 快速： 最快速的Python Framework
  • 開發快速

Requirements

  • Python 環境 : Python 3.6 +

  • 基本套件: (bash script)

    :::bash pip install fastapi # FastAPI pip install uvicorn[standard] # ASGI Server

撰寫第一支簡單的API

基本結構

在語法結構上，與Flask大同小異，透過 @app 裝飾子來定義 API路徑

from typing import Optional

from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"Hello": "World"}


@app.get("/users/{user_id}")
def read_user(user_id: int, q: Optional[str] = None):
    return {"item_id": user_id, "q": q}

啟動服務

與 Flask 不同的是，如果直接執行此 main.py 不會啟動服務。

需要透過 ASGI Server gvicorn 來啟用：

uvicorn main:app --reload
# uvicorn <檔名>:<app名稱>
# --reload 在專案更新時，自動重新載入，類似Flask的debug模式

啟用後在瀏覽器輸入 127.0.0.1:8000/users/8?q=minglunwu，將會得到如下的回傳結果: 

{"item_id":8,"q":"minglunwu"}

與 Flask 相同，當我們鍵入 127.0.0.1:8000/users/8?q=minglunwu， FastAPI會在定義好的裝飾子中，尋找符合路徑前綴: /users 的 function，也就是 read_user()

不同的是：

  • FastAPI 透過 型態提示 (Typing Hints) 來作為接受參數的依據，在定義參數時，需要一併定義參數型態。
  • 使用 @app 裝飾子來決定API適用的方法:
    • @app.post()
    • @app.get()
    • @app.put()
    • @app.delete()

型態提示(Typing Hints)

舉例來說： 在上圖的 read_user() 中，定義了兩個參數及對應的格式：

  • user_id: int格式
  • q: str格式

這個機制有兩個好處！

  1. 接收Request時

FastAPI在收到請求時，會自動檢查參數的格式是否符合條件

如果此時我們嘗試在瀏覽器中輸入 不符合規則的URL:

127.0.0.1:8000/users/error_type?q=minglunwu (將原本的 8 換成字串 error_type)

則我們會看到下列「有用」且「清楚」的錯誤訊息，不像Flask會噴出驚心動魄的錯誤訊息:

{"detail":[{"loc":["path","item_id"],"msg":"value is not a valid integer","type":"type_error.integer"}]}
